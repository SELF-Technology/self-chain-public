What SELF Does (Not You)‚Äã

When a user signs up through the SELF Super-App, we automatically:

  1. ‚úÖ Provision their private cloud instance
  2. ‚úÖ Deploy their SELF Chain node
  3. ‚úÖ Set up their private LLM
  4. ‚úÖ Configure their decentralized storage
  5. ‚úÖ Manage all infrastructure

Users get their cloud automatically. Developers don't provision anything.

As a developer, you:

  1. üî® Build apps that interact with user instances

How to Build Apps for SELF Users‚Äã

1. User Authentication‚Äã

Users log in with their SELF account:

// User authorizes your app
const auth = await SELF.authenticate({
  appId: 'your-app-id',
  permissions: ['ai:chat', 'storage:read']
});

2. Interact with User's Private LLM‚Äã

// Chat with user's private AI (with permission)
const response = await auth.ai.chat({
  message: "Help me plan my day",
  context: "productivity"
});

3. Store in User's Sovereign Storage‚Äã

// Save to user's private storage (with permission)
await auth.storage.save({
  key: 'app-data',
  value: { preferences: {...} },
  encrypted: true
});

What You Cannot Do‚Äã

‚ùå Access Without Permission‚Äã
